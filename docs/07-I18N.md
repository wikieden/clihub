# 国际化 (i18n)

## 设计原则

1. **极简自造**，不引 formatjs 等重量级库
2. **类型安全**，TypeScript 自动从 en.json 生成 key 类型
3. **缺 key 不崩**，自动 fallback 英语
4. **贡献门槛低**：翻译者只改 JSON

## locale 探测优先级

```
1. --lang <code>              # CLI flag
2. CLIHUB_LANG               # 环境变量
3. ~/.clihub/config.json     # 用户偏好
4. $LANG / $LC_ALL             # 系统
5. en                          # fallback
```

首次跑 TUI 弹「选语言」→ 写 `~/.clihub/config.json`。

## JSON 格式

key 用点分层，描述性命名：

```json
{
  "menu.main.title": "clihub main menu",
  "menu.main.skill": "Manage skill packs",
  "menu.main.mcp": "MCP servers",
  "doctor.cc_not_found": "Claude Code not found. Run 'clihub install-cc'",
  "skill.installed": "Installed skill {name}",
  "confirm.destructive": "This will modify {file}. Continue? (y/N)",
  "tool.claude-code.name": "Claude Code",
  "tool.claude-code.description": "Anthropic official AI coding CLI",
  "tool.codex.name": "Codex CLI",
  "tool.codex.description": "OpenAI Codex CLI"
}
```

zh-CN.json 对应：

```json
{
  "menu.main.title": "clihub 主菜单",
  "menu.main.skill": "管理 skill 包",
  "menu.main.mcp": "MCP server",
  "doctor.cc_not_found": "未找到 Claude Code。运行 'clihub install-cc' 安装",
  "skill.installed": "已装 skill {name}",
  "confirm.destructive": "此操作会修改 {file}，继续? (y/N)",
  "tool.claude-code.name": "Claude Code",
  "tool.claude-code.description": "Anthropic 官方 AI 编程 CLI",
  "tool.codex.name": "Codex CLI",
  "tool.codex.description": "OpenAI Codex CLI"
}
```

## API

```ts
import { t, setLocale, detectLocale } from '@clihub/core/i18n'

setLocale('zh-CN')
t('skill.installed', { name: 'tavily' })   // "已装 skill tavily"
t('unknown.key')                             // fallback: "unknown.key" + warn
```

`t()` 实现：

```ts
function t(key: I18nKey, params?: Record<string, string>): string {
  const dict = locales[currentLocale] ?? locales.en
  let s = dict[key] ?? locales.en[key] ?? key
  if (params) {
    for (const [k, v] of Object.entries(params)) {
      s = s.replaceAll(`{${k}}`, v)
    }
  }
  return s
}
```

## 类型安全

`gen-types.ts` 跑构建期：

```ts
export type I18nKey =
  | 'menu.main.title'
  | 'menu.main.skill'
  | 'menu.main.mcp'
  | ...
```

`t()` 签名 `t(key: I18nKey, ...)` 拼错 key 编译报错。

## 翻译完整性 lint

```bash
bun run i18n:lint
```

逻辑：

1. 用 en.json keys 为基准集合
2. 各语言 JSON 比对：缺 key 报 warn，多 key 报 error
3. CI 强制 lint 过

## 翻译流程

1. 改 `en.json` 加 key
2. `bun run i18n:gen-types` 更新类型
3. 提 PR 时附 zh-CN 翻译（核心维护者母语）
4. 其它语言留空，社区翻 PR 补齐
5. 翻译者只改对应 JSON，本地跑 `bun run i18n:lint` 自检

## LLM 辅助翻译

```bash
bun run i18n:translate ja           # 调 Claude API 翻缺的 key
```

输出建议 diff，人工 review 后提 PR。不接受未审 PR 直接合并机翻。

## 复数与变量

简化：不引 ICU MessageFormat。复杂场景写两个 key：

```json
{
  "skill.count.one": "1 skill installed",
  "skill.count.many": "{count} skills installed"
}
```

```ts
const key = count === 1 ? 'skill.count.one' : 'skill.count.many'
t(key, { count: String(count) })
```

## RTL / 阿语 / 希语

初期不专门优化，仅保证文本正确。后期视需要封装。

## 工具 description 的 i18n

ToolProvider / SkillManifest 不直接存翻译，而是 i18n key 引用：

```ts
{
  id: 'codex',
  name: 'tool.codex.name',
  description: 'tool.codex.description'
}
```

skill 包独立分发，i18n 必须自包含（manifest.json 内嵌）。

## 性能

- locales 全部内联到 bundle（每个 ~5-10KB JSON）
- 切语言不重启，运行时切 `currentLocale` 即可
- 总 10 语言 ≈ 100KB，可接受
